## [Stop Using JWTs for Browser Sessions](https://gist.github.com/samsch/0d1f3d3b4745d778f78b230cf6061452) ŌŁÉ’ĖÅ 8.0/10

A widely-discussed gist argues that JSON Web Tokens (JWTs) should not be used for browser-based user sessions due to security and complexity issues, while acknowledging their value for service-to-service communication. This debate highlights the widespread misuse of JWTs in web authentication, where simpler and more secure alternatives like session cookies exist, potentially leading to security vulnerabilities in many applications. JWTs are stateless and self-contained, but they cannot be easily revoked, often have long lifetimes, and expose session data to the client, making them less suitable for browser sessions compared to opaque session tokens.

hackernews ┬Ę dzonga ┬Ę Jun 16, 16:49 ┬Ę [Discussion](https://news.ycombinator.com/item?id=48558147)

**Background**: JWTs are a compact, URL-safe token format used for authentication and information exchange. They are often used in REST APIs and single-page applications. However, for browser-based sessions, traditional session cookies with server-side storage provide better security and revocation capabilities.
