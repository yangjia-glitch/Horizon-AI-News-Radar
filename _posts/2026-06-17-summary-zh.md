## [停止在浏览器会话中使用 JWT](https://gist.github.com/samsch/0d1f3d3b4745d778f78b230cf6061452) ⭐️ 8.0/10

一篇广受讨论的 Gist 文章认为，由于安全性和复杂性等问题，不应在基于浏览器的用户会话中使用 JSON Web 令牌（JWT），同时承认其在服务间通信中的价值。 这场争论凸显了 JWT 在 Web 认证中的广泛误用，而会话 cookie 等更简单、更安全的替代方案已经存在，这可能导致许多应用程序出现安全漏洞。 JWT 是无状态且自包含的，但难以撤销、通常生命周期较长，并将会话数据暴露给客户端，因此与不透明的会话令牌相比，不太适合浏览器会话。

hackernews · dzonga · 6月16日 16:49 · [社区讨论](https://news.ycombinator.com/item?id=48558147)

**背景**: JWT 是一种紧凑、URL 安全的令牌格式，用于身份验证和信息交换，常用于 REST API 和单页应用。然而，对于基于浏览器的会话，传统的带有服务器端存储的会话 cookie 提供了更好的安全性和撤销能力。
